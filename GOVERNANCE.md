# FreeLang 거버넌스

## 조직 구조

```
freelang-io (GitHub Organization)
├── freelang-compiler (메인 저장소)
├── freelang-docs (공식 문서)
├── freelang-playground (온라인 에디터)
├── freelang-examples (예제 모음)
└── freelang-community (커뮤니티 리소스)
```

## 역할

| 역할 | 권한 | 책임 |
|------|------|------|
| **Owner** | 전체 관리 | 전략/방향 결정 |
| **Maintainer** | 병합/릴리즈 | 코드 품질 관리 |
| **Contributor** | PR 제출 | 기능 개발/버그 수정 |
| **Community** | Issue 작성 | 피드백/보고 |

## 의사결정 프로세스

1. **RFC (Request for Comments)**: 큰 변경 사항은 먼저 제안
2. **토론**: 커뮤니티 의견 수렴
3. **투표**: Maintainer 투표 (2/3 이상)
4. **구현**: 승인된 항목만 진행
