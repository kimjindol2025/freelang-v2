❯ 그래서 배울점은?                                                              

● PyFree에서 배울 점

  1. 테스트가 많다고 품질이 좋은 건 아니다

  - 640개 테스트 통과라고 했지만, 실제 파이프라인 end-to-end 테스트가 없었음
  - mock/시뮬레이션 테스트만으로는 실제 동작을 보장 못함
  - FreeLang v2 교훈: 우리가 한 것처럼 .fl 파일을 직접 컴파일+실행하는 통합
  테스트가 핵심

  2. "동작하는 척"을 경계해야 한다

  - run-selfhosting.ts가 순수 TypeScript인데 "셀프호스팅 성공"이라 주장
  - 파이프라인을 우회하면 셀프호스팅이 아님
  - 교훈: 셀프호스팅은 반드시 자기 언어로 작성된 코드가 자기 파이프라인을 
  통과해야 의미가 있음

  3. 레지스터 vs 스택 머신 설계 선택

  - PyFree: 레지스터 머신 (256개 레지스터) → 할당 복잡도 높음 → 버그 발생
  - FreeLang v2: 스택 머신 → 단순하고 버그 적음
  - 교훈: 복잡한 설계보다 단순하고 검증 가능한 설계가 셀프호스팅에 유리

  4. 문서/주장보다 실행 결과가 진실

  - README에 "Phase 1 Runtime 100%", "640/640 통과" 써있어도
  - print(42) 하나 제대로 못 돌아가면 의미 없음
  - 교훈: FreeLang v2처럼 실제 실행 → 출력 확인 → 버그 수정 반복이 정직한 방법

  5. FreeLang v2가 잘한 것

  - 62/67 셀프호스팅 테스트가 실제로 동작함
  - 버그를 하나씩 찾아서 고침 (length, push, continue, member assign 등)
  - 예제 파일(data-structures.fl, self-hosting-demo.fl)이 실제로 올바른 결과
  출력

  한 줄 요약: 화려한 숫자보다 print(42) → 42 하나가 더 가치있다.

// src/parser.ts (또는 해당 파서 파일)                                          
                                                                                
private is_identifier_token(token: Token): boolean {                            
    const type = token.type;                                                    
    // 일반 식별자뿐만 아니라, 변수명으로 쓰일 가능성이 높은 타입 키워드들을    
허용                                                                            
    return (                                                                    
        type === TokenType.IDENTIFIER ||                                        
        type === TokenType.TYPE_INT ||                                          
        type === TokenType.TYPE_STRING ||                                       
        type === TokenType.TYPE_MAP ||                                          
        type === TokenType.TYPE_LIST ||                                         
        type === TokenType.TYPE_ANY                                             
    );                                                                          
}                                                                               
                                                                                
private parse_identifier_name(): string {                                       
    const token = this.peek();                                                  
    if (this.is_identifier_token(token)) {                                      
        this.advance(); // 토큰 소모                                            
        return token.value;                                                     
    }                                                                           
    throw new Error(`Expected identifier but found ${token.type}`);             
}
